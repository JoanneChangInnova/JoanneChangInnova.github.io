Hosting for GitHub Pages Link to heading

  1. create a Github repository
- user site: https://<username>.github.io/
- project sites: https://<username>.github.io/<repository>

You can only create one user site for each account on GitHub. Project sites are unlimited.

  1. edit config.toml/yml: add baseURL

  2. upload files to repository:

cd public
git submodule add -f <repo-url> public
git add .
git branch -M main
git commit -m "first commit"
git push -u origin main
  1. start building

Hosting for AWS S3 Link to heading

  1. create S3 bucket
# add secret access key and region of your account (create one in Security Credentials)
aws configure 
# create bucket
aws s3 mb s3://bucket-name
# check all buckets
aws s3 ls

  1. enable static website hosting

  2. permission > edit bucket policy

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicReadGetObject",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::<bucket-name>/*"
        }
    ]
}

this will allow public read and get all files in your bucket

  1. upload files
aws s3 sync . s3://my-bucket/
  1. your website url is under properties > Static website hosting
